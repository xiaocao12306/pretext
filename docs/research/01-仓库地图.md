# Pretext 仓库地图

这张地图服务于 [[01-项目总览]] 和后续模块卡：先把“仓库有哪些层”讲清楚，再进入每个文件。

## 一句话地图
Pretext 仓库可以先粗分为五层：
- 核心引擎：`src/`
- 浏览器页面与 demo：`pages/`
- 自动化与校验脚本：`scripts/`
- 语料与快照数据：`accuracy/`、`benchmarks/`、`corpora/`
- 顶层文档与发布配置：`README.md`、`DEVELOPMENT.md`、`package.json`、`tsconfig*.json`

## 目录分层

### 1. 顶层产品与工程元数据
- `README.md`：公开 API、核心卖点、限制条件
- `DEVELOPMENT.md`：开发命令、校验命令、当前 source of truth
- `package.json`：发布入口、脚本矩阵、导出面
- `tsconfig.json` / `tsconfig.build.json`：开发与发布时的 TypeScript 编译边界
- `STATUS.md` / `RESEARCH.md` / `TODO.md`：当前性能准确性快照、历史研究日志、优先级

### 2. `src/` 核心引擎
- `src/layout.ts`：公共 API 入口，连接 analyze / measure / line-break / bidi
- `src/analysis.ts`：文本归一化、`Intl.Segmenter` 分词、glue/URL/数字/CJK/Arabic 等预处理规则
- `src/measurement.ts`：canvas 度量、segment metrics cache、emoji correction、engine profile
- `src/line-break.ts`：真正的行走算法，服务 `layout()`、`walkLineRanges()`、`layoutNextLine()`
- `src/bidi.ts`：仅 rich path 消费的 bidi level 元数据
- `src/layout.test.ts` / `src/test-data.ts`：小而稳定的持久测试语料与不变量

### 3. `pages/` 浏览器页面
- 诊断/回归页面：`accuracy.ts`、`benchmark.ts`、`corpus.ts`、`probe.ts`、`gatsby.ts`
- 公共辅助：`diagnostic-utils.ts`、`report-utils.ts`
- HTML 入口：与上述页面配套的 `.html`
- demos：`pages/demos/**/*`，包括 editorial、accordion、bubbles、dynamic-layout、rich-note、masonry、justification-comparison

### 4. `scripts/` 自动化与产物生成
- 浏览器校验：`accuracy-check.ts`、`benchmark-check.ts`、`pre-wrap-check.ts`、`probe-check.ts`
- 语料相关：`corpus-check.ts`、`corpus-sweep.ts`、`corpus-font-matrix.ts`、`corpus-taxonomy.ts`、`corpus-representative.ts`
- Gatsby：`gatsby-check.ts`、`gatsby-sweep.ts`
- 支撑：`browser-automation.ts`、`report-server.ts`
- 发布/站点：`package-smoke-test.ts`、`build-demo-site.ts`

### 5. 数据与基准资产
- `accuracy/*.json`：浏览器 sweep 原始快照
- `benchmarks/*.json`：性能快照
- `corpora/*`：长文语料、来源、taxonomy、status、representative canaries

## 文件规模观察
- 最大核心文件是 `src/line-break.ts`（1056 行）与 `src/analysis.ts`（1007 行），说明断行语义和预处理规则是复杂度重心
- 最大页面文件是 `pages/corpus.ts`（1134 行）、`pages/demos/editorial-engine.ts`（968 行）、`pages/demos/dynamic-layout.ts`（922 行）、`pages/gatsby.ts`（836 行）
- 最大脚本文件是 `scripts/browser-automation.ts`（649 行）与 `scripts/corpus-font-matrix.ts`（515 行）
- 这意味着研究顺序应优先关注：`src/` 核心引擎，其次是 corpus/gatsby/browser automation 这些“验证与诊断重心”

## 研究顺序建议
1. 先完成 [[01-项目总览]] 与顶层工程面
2. 再进入 `src/` 的模块卡与实现卡
3. 再研究 `pages/` 的验证页面与 demo 页面
4. 最后把 `scripts/`、`corpora/`、快照数据接回到功能与风险层
