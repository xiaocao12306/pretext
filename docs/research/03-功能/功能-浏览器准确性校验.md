# 功能：浏览器准确性校验

入口：
- `pages/accuracy.ts`
- `scripts/accuracy-check.ts`（后续脚本研究补充）

相关模块：[[模块-浏览器验证页面]]、[[模块-测试与共享测试数据]]

## 功能目标
验证 Pretext 的高度预测是否与真实 DOM 布局一致，并把结果沉淀为可回归比较的浏览器快照。

## 页面主链
1. 读取共享语料：`TEXTS`、`SIZES`、`WIDTHS`
2. 对每个 `font × size × width × text`：
   - 用 DOM `div` 获取真实高度
   - 用 `prepareWithSegments()` + `layout()` 计算预测高度
3. 如果 mismatch：
   - 用 `layoutWithLines()` 取我方行
   - 用 DOM range extractor 取浏览器行
   - 记录 per-line diagnostic
4. 输出 summary + mismatches + 可选 full rows

## 重要细节

### 共享 sweep 维度
页面内定义 4 个 font family，结合 `src/test-data.ts` 的 8 sizes、8 widths、30 texts，形成大矩阵 sweep。

### mismatch 后才升级诊断
正常路径只比较 block height；只有 mismatch 时才做更贵的 per-line line extraction 与 `layoutWithLines()` 对照。

### 报告模式
- 默认把结果挂到 `window.__ACCURACY_REPORT__`
- 小报告走 hash navigation state
- 大报告可走 `reportEndpoint` POST

## 当前判断
- 这是“广覆盖回归门”，不是“最深诊断工具”
- 它依赖共享测试语料，因此和 benchmark 页共享一部分输入基线
