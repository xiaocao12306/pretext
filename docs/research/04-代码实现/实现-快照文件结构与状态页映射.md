## 这张卡关注什么
这里不看页面或脚本本身，而看“脚本跑完以后，结果以什么文件结构落地，并怎样再被状态页消费”。

## 1. accuracy 快照是“全量 raw row + 环境指纹”

`accuracy/*.json` 的结构很稳定：
- `status`
- `environment`
- `total`
- `matchCount`
- `mismatchCount`
- `mismatches`
- `rows`

关键点：
- `mismatches` 让状态摘要读取很便宜
- `rows` 保留了全量回放材料
- `environment` 让同一个 `7680/7680` 不再脱离具体浏览器环境

所以它不是纯 dashboard，也不是最小摘要，而是一种“可压缩但可回看的 raw snapshot”。

## 2. benchmark 快照把结果分成四个成本层

`benchmarks/*.json` 当前至少分成：
- `results`
- `richResults`
- `richLongResults`
- `corpusResults`

这不是随意分组，而是对应四种不同问题：
- cold prepare / hot layout / DOM resize 的整体基线
- rich API 的短文本成本
- rich API 的长文成本
- 每个 corpus 的结构统计与 prepare/layout 分拆成本

其中 `corpusResults` 特别有价值，因为它把：
- `analysisSegments`
- `segments`
- `breakableSegments`
- `lineCount`

与
- `analysisMs`
- `measureMs`
- `prepareMs`
- `layoutMs`

放在了一起，方便把“文本结构”和“成本来源”直接关联。

## 3. `sources.json` 是 corpus 的 canonical schema

脚本层会把 `sources.json` 当作注册表，而不是简单数据文件。

它承载三类实现含义：
- 身份：`id`、`title`、`language`、`direction`
- 运行参数：`font_family`、`font_size_px`、`line_height_px`、`default_width`、`min_width`、`max_width`
- 研究 provenance：`source_url`、`acquired_via`、`notes`、`characters`、`lines`

所以 corpus 脚本不是在自己发明 sweep 范围，而是在消费这份 schema。

## 4. `corpora/STATUS.md` 是人工压缩视图，不是原始事实源

它的实现角色是：
- 把官方 gate、产品型 canary、长文 canary、fine sweep、font matrix 分区展示
- 把值得长期记住的 mismatch field 压缩成人类可读笔记
- 给出 recompute 命令

这意味着它依赖但不等于 raw snapshot。

理解这点很重要：
- `accuracy/*.json` / `representative.json` 更像 source data
- `corpora/STATUS.md` 更像 curated dashboard

## 5. `representative.json` 是“可比较最小快照”

文件结构：
- `generatedAt`
- `corpora`
- `widths`
- `browsers`
  - `environment`
  - `rows`

每条 `rows` 记录都带：
- `corpusId`
- `title`
- `language`
- `direction`
- `width`
- `contentWidth`
- `font`
- `lineHeight`
- `predictedHeight`
- `actualHeight`
- `diffPx`
- `predictedLineCount`
- `browserLineCount`

它的核心实现价值是：
- 比完整 sweep 小
- 比人工状态页精确
- 足够支持版本间 anchor 对比

## 6. taxonomy 文件不是结果表，而是解释表

`corpora/TAXONOMY.md` 的作用不是存放某次 sweep 的输出，而是稳定 category 定义与 steering rules。

因此它和 `corpus-taxonomy.ts` 的关系是：
- 脚本给出启发式分类结果
- 文档给出分类语义、适用边界、下一步动作

这是“结果 → 语言 → 决策”的最后一层映射。

## 当前判断
- Pretext 的状态页系统其实是多层视图：
  - raw snapshot
  - representative snapshot
  - curated status page
  - taxonomy vocabulary
- 这层分离做得相当克制。它没有试图让任何一个文件既做原始数据库，又做人类仪表盘，又做研究日志
